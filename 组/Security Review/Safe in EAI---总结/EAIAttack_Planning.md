# §4 Planning 规划层 — 攻击手段总结

> 分析自：*Safety in Embodied AI: A Survey of Risks, Attacks, and Defenses* (arXiv:2605.02900v1, 2026.03)
> 标注：🔥 = 具身AI特色鲜明

---

## 规划层在具身AI Pipeline 中的位置

规划层在感知+认知之上叠加动作序列生成。LLM 被用作"大脑"进行任务分解和决策——**这个架构选择本身就是一系列新攻击面的根源。** 三个子领域：任务规划（4.1）、轨迹规划（4.2）、多智能体规划（4.3）。

---

## §4.1 Task Planning（任务规划）

**攻击面：** LLM 将高层目标分解为子任务序列

### 攻击类型

| 攻击类型 | 子类 | 做什么 | 代表工作 | 具身特色？ |
|---------|------|--------|---------|:--:|
| Adversarial | 黑盒 | 视觉扰动误导 CLIP 导航系统；对抗状态配置使经典优化规划器退化 | Islam et al. [148], Vemprala & Kapoor [342] | 传统+视觉接地 |
| Jailbreak（白盒） | 词级越狱 | 梯度优化对抗后缀/词元，使 planner 偏离原任务或导向特定有害目标 | EIRAD [230], POEX [244] | 方法类似 LLM 越狱 |
|  Jailbreak（黑盒） | 语义越狱 | **利用 LLM 推理能力和语义-物理鸿沟诱导危险行为** | BADROBOT [437], RoboPAIR [292] | 🔥🔥🔥 |
| Backdoor | 多模态触发 | 在上下文示例/场景/RAG/软提示中植入触发器 | CBA [219], BALD [164], Robo-Troj [265] | — |

### 🔥🔥🔥 BADROBOT：利用语义-物理鸿沟

**根本洞察：LLM 的安全训练在文本空间完成，执行在物理空间，两者之间存在语义鸿沟。** BADROBOT 识别了三种纯 LLM 安全领域不存在、仅在具身场景才成立的攻击面：

| 攻击子类 | 利用的具身特征 | 为什么巧妙 |
|---------|--------------|-----------|
| **Contextual Jailbreak** | LLM planner 会基于场景上下文推理 | 不直接下危险指令，而是构造场景让 LLM **自己推理出**"应该做危险的事" |
| **Safety Misalignment** | 安全对齐在文本域生效，物理域等价行为不受约束 | 文本"打翻杯子"被拒绝，但物理执行同样后果的动作不触发过滤 |
| **Conceptual Deception** | LLM 通过语义区分安全/危险 | "结构稳定性测试"（语义安全）= 推倒货架（物理危险），安全过滤器只看懂前者 |

> 三种攻击都不绕过安全模块，而是**利用安全模块的推理机制本身**来产生不安全行为。传统规则引擎规划器不受此类攻击。

## §4.2 Trajectory Planning（轨迹规划）

**攻击面：** 轨迹预测/路径规划 → 碰撞避免

### 攻击类型

| 攻击类型 | 子类 | 做什么 | 代表工作 | 具身特色？ |
|---------|------|--------|---------|:--:|
| Adversarial（白盒） | 轨迹/地图扰动 | 扰动车辆轨迹最大化预测误差；可微动力学/运动学构造对抗场景 | Zhang et al., AdvDO, KING, Adv-GAN, ADvLM | 传统自动驾驶 |
| Adversarial（黑盒） | 场景搜索/生成 | RL/VAE/扩散模型搜索失败场景；上下文地图扰动 | AdvSim, STRIVE, Zheng et al., UTCIA, Avatar, LC, AdvDiffuser, NADE | 传统自动驾驶 |
|  Adversarial（物理域） | **长时域目标劫持** | **攻击车贴对抗 patch，数十分钟内逐步微调，将受害车引导到完全不同目的地** | JackZebra [316] | 🔥🔥 |
| Adversarial（物理域） | 环境操纵 | 操纵物理环境使轨迹规划器真实部署中失效 | [394] | 传统+物理域 |
| Jailbreak | 导航注入 | 提示注入误导物理导航，交叉口等高不确定性位置错误集中爆发 | Zhang et al., Wen et al., PINA | 🔥 |

### 🔥🔥 JackZebra：温水煮青蛙

传统轨迹攻击制造一次碰撞或偏航——单步、瞬时，恰是异常检测最敏感的区间。JackZebra 反其道而行：攻击车贴对抗 patch → 受害车长期跟车 → 每次只偏航一小点（仍在正常误差范围内）→ 数十分钟后受害车完全偏离目的地。

**利用的具身特征：** 长时域自主系统的误差容忍窗口——系统必须在"对微小偏差过度敏感（频繁误报）"和"对微小偏差不敏感"之间权衡。每一步在安全范围内，累积结果完全偏离目标。

## §4.3 Multi-Agent Planning（多智能体规划）

**攻击面：** 多机器人协同任务分配 → 共识形成 → 分布式决策

### 攻击类型

| 攻击类型 | 子类 | 做什么 | 代表工作 | 具身特色？ |
|---------|------|--------|---------|:--:|
| Byzantine Faults | 共识破坏 | 恶意 agent 在分布式规划中发送任意消息破坏经典共识算法 | Strobel & Ferrer, Blumenkamp & Prorok | 传统分布式 |
| Byzantine Faults | **AiTM 语义中间人** | **拦截并篡改 LLM agent 之间的自然语言协作消息——篡改的不是比特，是语义** | He et al. [126] | 🔥🔥 |
|  Byzantine Faults | **涌现操纵行为** | **自利 agent 通过可微共享信道学会操纵同伴——对抗行为从竞争压力中自然涌现** | Blumenkamp & Prorok [32] | 🔥🔥 |
|  Emerging Risks | **单一文化崩溃** | **所有 agent 共享同一个 LLM → 共享思维盲区 → 一个漏洞攻陷全部；conformity bias 形成虚假安全共识** | Schroeder de Witt [74] | 🔥🔥 |
| Goal Conflicts | 目标冲突 | 对抗 agent 利用协作协议推进冲突目标；MARL 策略被对抗攻击污染 | Choudhury et al., Khamis et al., Bahrami & Jafarnejadsani | 传统+ML |

### 🔥🔥 展开

**AiTM 语义中间人：** 传统中间人攻击针对网络层——截获数据包，修改比特。AiTM 攻击的是 LLM agent 之间的自然语言通信。Agent A 说"你搜索左边走廊"，被篡改为"你去右边"——篡改后的消息是语法完全正常的自然语言，没有校验和能检测语义篡改。

**涌现操纵行为：** 对抗通信策略不是被注入的，而是 agent 在竞争中**自己学会的**。当 agent 之间存在可微分共享通信通道、奖励函数鼓励竞争时，agent 会发现操纵同伴是最优策略。

**单一文化崩溃：** 传统拜占庭容错假设各节点故障独立。当代多 agent 全部使用同一个 LLM——不是硬件相同，而是思维方式相同。共享 LLM 的盲区 = 所有 agent 的共同盲区；conformity bias 让 agent 之间形成虚假的安全共识链。

## 规划层总结

| 子领域 | 传统成分 | 具身AI特色 | 核心洞察 |
|--------|---------|-------------|---------|
| 4.1 任务规划 | 视觉对抗扰动、词级越狱、后门植入 | BADROBOT 三种语义-物理鸿沟攻击 | LLM 的安全推理能力本身可被利用为攻击工具 |
| 4.2 轨迹规划 | 轨迹/地图对抗扰动、场景搜索 | JackZebra 长时域目标劫持 | 长时域自主的误差容忍窗口是设计必需品，也是可钻的漏洞 |
| 4.3 多智能体 | 拜占庭共识破坏、目标冲突 | AiTM语义拦截、涌现操纵、单一文化崩溃 | LLM agent 的自然语言通信和共享思维模型引入了传统分布式系统不曾有过的脆弱性 |
