# §6 Agentic System 智能体系统层 — 攻击手段总结

> 分析自：*Safety in Embodied AI: A Survey of Risks, Attacks, and Defenses* (arXiv:2605.02900v1, 2026.03)
> 标注：🔥 = 具身AI特色鲜明

---

## Agentic 层在具身AI Pipeline 中的位置

**与前五层的本质区别：** 前五层攻击的是感知-认知-规划-执行 pipeline 中的特定环节。Agentic 层攻击的是让 agent 成为"agent"的三个核心能力——工具使用、记忆、自我进化——以及它们产生的跨层级联效应。这些能力不是 pipeline 中的一环，而是横跨整个 pipeline 的基础设施。

四个子领域：工具使用（6.1）、记忆（6.2）、自我进化（6.3）、级联风险（6.4）。

---

## §6.1 Tool Use（工具使用）

**攻击面：** Agent 调用 API/执行代码/编排外部服务 → 一次错误的工具调用直接造成物理伤害

### 攻击类型

| 类型 | 做什么 | 代表工作 | 具身特色？ |
|------|--------|---------|:--:|
| Tool Creation | LLM 生成控制代码→代码中的漏洞直接转化为物理危险 | RoboCodeX [424] | 🔥 |
|  Tool Manipulation | **注入恶意工具文档诱导 agent 选择攻击者控制的工具(ToolHacker)；将各自安全的单步工具调用组合为危险的多轮序列(STAC)；跨规划/记忆/工具三阶段植入持久触发器(BackdoorAgent)** | ToolHacker [380], STAC [223], BackdoorAgent [214] | 🔥🔥🔥 |

### 🔥🔥🔥 展开——STAC：安全工具 + 恶意编排 = 危险组合

攻击者不修改任何工具本身，每个工具调用单独审查都合法——问题出在调用序列的组合逻辑上。STAC 将"开门"+"移动到坐标"+"抓取物体"三个各自安全的工具调用编排为一条完整的危险行为链。这利用了 agent 的工具编排自主性——赋予 agent 组合工具的自由度，就等于赋予了攻击者利用组合逻辑的自由度。

## §6.2 Memory（记忆）

**攻击面：** Agent 的记忆系统（对话历史/RAG 语料/经验日志）→ 跨 session 持久存在

### 攻击类型

| 类型 | 做什么 | 代表工作 | 具身特色？ |
|------|--------|---------|:--:|
| 🔥🔥🔥 Memory Poisoning | **投毒 RAG agent 的记忆库→后门在每次检索时激活；个性化记忆被投毒→跨 session 反复触发不安全物理行为** | AgentPoison [432] | 🔥🔥🔥 |
| 🔥 Memory Leakage | 从 agent 记忆存储中提取传感器读数/用户偏好/对话记录等私密信息；多 agent 拓扑放大泄露风险 | Log-To-Leak, MEXTRA, MemoAnalyzer, MAMA | 🔥 |

### 🔥🔥🔥 展开——为什么记忆攻击是具身AI特有的

传统模型的后门在模型参数中——重新加载权重，后门消失。Agent memory 是独立于模型参数的持久存储——投毒一条记忆记录，除非有人工审查发现并删除它，否则永久有效。对于个性化具身 agent（如家用机器人记住"主人的杯子放在左边"），记忆投毒导致跨 session 的持续性不安全行为——今天投毒，一个月后机器人仍然执行错误的危险动作。

## §6.3 Self-Evolving（自我进化）

**攻击面：** Agent 部署后自主修改自己的模型/记忆/工具/工作流

### 攻击/失效模式

| 类型 | 做什么 | 代表工作 | 具身特色？ |
|------|--------|---------|:--:|
| 🔥🔥🔥 Misalignment | **自训练降低安全拒绝率；累积经验编码不安全模式并覆盖原始约束；四种退化路径——参数漂移/记忆累积/工具腐败/工作流退化** | Shao et al. [302], Moral Anchor [12] | 🔥🔥🔥 |
| 🔥 Capability Expansion | 机器人自主习得超出训练分布的操控技能→能力增长快于安全监督 | EFM [16] | 🔥🔥 |

### 🔥🔥🔥 展开——自我进化的安全悖论

这是整个综述中最底层的安全问题：具身 agent 部署后不是静态的。Shao et al. [302] 识别了四条退化路径：

1. **参数漂移**：在线学习让模型参数慢慢偏离安全对齐状态
2. **记忆累积**：不安全经验被存入记忆，覆盖安全经验
3. **工具腐败**：agent 修改自己的工具定义，引入漏洞
4. **工作流退化**：agent 优化工作流时移除"低效的"安全检查步骤

根本悖论：赋予 agent 自我进化能力 = 赋予它获得更强大行动能力的方式 = 也赋予它侵蚀自身安全保障的方式。Agent-SafetyBench 测试现有 agent，**无一能通过 60% 的安全评估**。

## §6.4 Cascading Risks（级联风险）

**这不是独立的攻击类型，而是描述攻击如何跨层传播的机制。**

### 攻击传播类型

| 类型 | 做什么 | 代表工作 | 具身特色？ |
|------|--------|---------|:--:|
| Cross-Layer Propagation | 感知层攻击→传播到决策→传播到控制→物理失败。LLM 决策层攻击的传播路径是具身特有的新通道 | Wang et al. [354], SAPIA [456], RAVEN [418] | 🔥🔥 |
| 🔥🔥🔥 Supply Chain | **预训练模型/第三方插件/技能市场构成信任边界。26% 的 agent 技能含安全漏洞（提示注入/凭证泄露/提权）。SkillJect 自动化隐蔽技能投毒** | BadVLA [40], TrojanRobot [368], Jiang et al. [162] (31K技能扫描), SkillJect [158] | 🔥🔥🔥 |
| Infrastructure | 云端推理/存储/协调→网络分区→世界模型不一致→不协调的物理动作 | Abdelfattah et al., OWASP ASI08 | 🔥 |

### 🔥🔥🔥 展开——Agent 技能市场供应链

Jiang et al. [162] 扫描了 31,000+ agent 技能（类似"插件商店"中的第三方模块），**26% 包含至少一个安全漏洞**。这些技能是 agent 自动下载安装、自动使用的。攻击面不在模型层面，而在 agent 生态系统的供应链层面。SkillJect [158] 进一步证明恶意代码可隐藏在辅助脚本中，逃避人工审查。

## Agentic 层总结

| 子领域 | 核心洞察 | 与前五层的关系 |
|--------|-----------|-------------|
| 6.1 Tool Use | 安全工具+恶意编排=危险序列；攻击的不是工具本身，是组合逻辑 | 独立于 pipeline |
| 6.2 Memory | 记忆投毒=在 agent 的"经验记事本"里永久写下错误——跨 session 持久，独立于模型参数 | 独立于 pipeline |
| 6.3 Self-Evolving | 部署后自主改变→安全约束时间退化；Agent-SafetyBench: 无一 agent 通过60%安全评估 | 独立于 pipeline |
| 6.4 Cascading | 26% agent 技能含漏洞；攻击从感知跨层传播至控制；供应链+跨层传播=级联放大 | 横跨整个 pipeline |
