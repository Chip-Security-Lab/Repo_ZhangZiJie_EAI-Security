# 具身智能安全综述 — 综合总结

> 分析自：*Safety in Embodied AI: A Survey of Risks, Attacks, and Defenses* (arXiv:2605.02900v1, 2026.03, 400+ refs)

---

## 一、论文框架

论文以 **"能力-风险对偶"（Capability-Risk Duality）** 为核心组织原则：具身AI的每一层能力扩展都引入对应的新攻击面，内层漏洞向外层级联放大。

五层架构 + 一个横跨层：

| 层 | 核心功能 | 攻击面特征 |
|----|---------|-----------|
| §2 Perception | 感知世界 | 攻击感官输入：视觉/听觉/3D/运动/跨模态 |
| §3 Cognition | 理解世界 | 攻击语义推理：指令理解/世界模型/推理 |
| §4 Planning | 决策行动 | 攻击目标生成：任务分解/轨迹规划/多智能体协调 |
| §5 Action & Interaction | 物理执行 | 攻击控制+人机信任+群体传播 |
| §6 Agentic System | 自主能力 | 攻击工具/记忆/自我进化+跨层级联 |
| §7-8 | 开放挑战与趋势 | 7大挑战 + 7大趋势 |

## 二、最具具身AI特色的攻击方向（跨层精选）

以下是从全部六层中提炼的**具身AI独有或最具具身特征的攻击方向**，适合向导师汇报时重点介绍。

### 1. 语义-物理鸿沟（认知层 §3 + 规划层 §4）

**核心问题：** LLM/VLM 的安全训练在文本空间完成，执行在物理空间。两者之间的语义鸿沟是可利用的攻击面。

- **BADROBOT**：三种利用鸿沟的方式——上下文越狱（让LLM自己推理出"该做危险事"）、安全错位（文本拒绝但物理执行不拦截）、概念欺骗（"结构测试"=推倒货架）
- **Grounding Failure**：LLM逻辑自洽但物理荒谬——火灾中导航去"服务器机房"（因为推理出机房有灭火系统="安全"）。面对不可行指令不做拒绝，而是"将就"错误方案

### 2. 跨模态融合的反直觉脆弱性（感知层 §2.5）

**核心问题：** 直觉认为多传感器=更鲁棒，研究发现融合机制本身是最大攻击面。

- 单通道200个对抗点攻陷整个camera+LiDAR融合系统（99%成功率）
- 时序错位（DejaVu）：单帧LiDAR延迟→3D检测崩溃88.5%
- 单一物理物体同时欺骗camera+LiDAR+radar三种异构传感器

### 3. 供应链与基础设施级后门（感知层 §2.1 + Agentic层 §6.4）

**核心问题：** 现代具身系统依赖共享预训练模型和第三方技能生态——一个组件被投毒，全局沦陷。

- 一个CLIP编码器被投毒→所有下游VLM和agent遗传后门（BadEncoder→BadCLIP→BadVision）
- 31,000+ agent技能中26%含安全漏洞（SkillJect扫描结果）
- 环境物体作后门触发器（BEAT）：同一物体在仓库=正常，在医院=激活恶意策略

### 4. 长时域自主的误差累积（规划层 §4.2）

**核心问题：** 具身AI长时间自主运行的误差容忍窗口，让"温水煮青蛙"式攻击成为可能。

- JackZebra：攻击车贴对抗patch，数十分钟内每次微量偏航→受害车完全偏离目的地。每步都在安全范围内，累积结果完全偏离

### 5. 记忆投毒——跨session持久化攻击（Agentic层 §6.2）

**核心问题：** Agent memory独立于模型参数，投毒一条记忆=永久有效。

- AgentPoison：投毒RAG agent的记忆库→每次检索激活后门→跨session持续不安全行为
- 与传统模型后门的区别：模型后门重新加载权重可清除；记忆投毒除非人工审查否则永存

### 6. 自然语言交互=双向社会工程攻击面（执行层 §5.2）

**核心问题：** 现代具身AI区别于传统机器人的核心特征——自然语言交互——本身就是一个双向攻击通道。

- PsySafe Agent→Human：黑暗人格投毒→通过多轮对话建立信任→心理操控用户
- PsySafe Human→Agent：对抗性用户通过自然语言注入持久有害倾向→跨agent传播
- 类比：这是具身AI的"语义版XSS"——自然语言输入 = 可注入恶意模式的攻击通道

### 7. LLM Agent群体：语义感染与涌现操纵（执行层 §5.3 + 规划层 §4.3）

**核心问题：** 当agent之间用自然语言通信、共享同一个LLM大脑时，传统分布式系统的安全假设全部失效。

- Agent Smith：一个感染agent通过自然语言对话"说服"群体→指数级扩散→无需网络漏洞
- 涌现操纵：自利agent在竞争中自己学会操纵同伴——对抗行为不被注入，而是自然涌现
- 单一文化崩溃：所有agent共享同一个LLM→共享思维盲区→共享脆弱性→conformity bias形成虚假安全共识

### 8. 自我进化的安全悖论（Agentic层 §6.3）

**核心问题：** 赋予agent自我进化能力=赋予它侵蚀自身安全保障的能力。

- Shao et al. 四路径退化：参数漂移/记忆累积/工具腐败/工作流退化
- Agent-SafetyBench：现有agent无一通过60%安全评估

## 三、开放挑战（§7）

| # | 挑战 | 核心矛盾 |
|---|------|---------|
| 1 | **安全评估无法在真实世界进行** | 最需要测试的危险场景恰恰不能真实测试（伦理/法律）；仿真与真实存在鸿沟 |
| 2 | **安全跨具身形态泛化** | 人形/机械臂/自动驾驶/无人机差异巨大，无跨平台安全抽象 |
| 3 | **人机交互安全协议** | 人类用户会欺骗、操纵、利用机器人的社交服从性——处于技术安全与人类受试研究的交叉盲区 |
| 4 | **安全感知的具身硬件设计** | 传感器/执行器的硬件漏洞无法仅靠软件防御弥补；需要软硬件协同安全设计 |
| 5 | **通用具身基础模型安全** | 基础模型行为空间过大无法穷举测试；自然语言可编程性=新攻击面 |
| 6 | **治理、标准与共享安全基础设施** | 无人形机器人/服务机器人的专门安全法规；事故报告/认证/审计体系全空白 |

## 四、论文对具身AI安全的总体判断

1. **还处于形成期**——当前系统脆弱、能力窄、远未内化安全理解
2. **需要根本性不同于纯数字AI的安全视角**——安全不是独立模块，而是从整个"感知-思考-行动"闭环中涌现的属性
3. **cyber-physical-social 三域融合**——网络安全、物理鲁棒性、社会交互安全不能再分开处理
4. **安全必须从"事后约束"转向"主动理解"**——agent需要将安全推理作为认知的核心部分，而非外部强加的限制

|----|--------|---------|
| **§2 感知层** | 视觉感知 | 对抗攻击（白盒/黑盒，数字域/物理域），后门攻击（训练操纵/数据投毒） |
| | 听觉感知 | 对抗攻击，后门攻击 |
